type连接类型

type官网全程“join type”，可以理解为一种数据库引擎查找表数据的一种方式，也可以称呼为访问lei型。经常遇到的六种类型分别是all，index，range，ref，eq_ref,const.从左到右效率依次增强。

all

这就是所谓的全表扫描，非常耗时且低效。我们应该避免这种类型的出现。

上面就是典型的全表扫描。

index

索引全扫描。

上面这种Extra是using index 索引覆盖，就是通过索引查询就能获取到所有结果，无需回表。

这种就进行了回表查询。

range

索引范围扫描，常见于使用 =, <>, >, >=, <, <=, IS NULL, <=>, BETWEEN, IN()或者like等运算符的查询中。

ref

这种情况出现就是虽然使用到了索引，但是索引的值不唯一。会进行索引小范围的查找。

ref_eq

出现在要连接过个表的查询计划中，驱动表只返回一行数据，且这行数据是第二个表的主键或者唯一索引，且必须为not null，唯一索引和主键是多列时，只有所有的列都用作比较时才会出现eq_ref

const

使用唯一索引或者主键，返回记录一定是1行记录的等值where条件时，通常type是const。其他数据库也叫做唯一索引扫描
